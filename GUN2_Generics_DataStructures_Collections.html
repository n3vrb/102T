1 Generics (Ch 19) ⚡ Sınav Projesi Temeli

Neden Generics?

  • Compile-time'da type safety sağlar
  • Cast yapmaya gerek kalmaz
  • Primitive type kullanılamaz → wrapper class kullan
🔴 Sınavda kesin çıkan: ArrayList<int> → COMPILE ERROR!
Doğrusu: ArrayList<Integer> — int yerine Integer, double yerine Double, boolean yerine Boolean. 
// ❌ HATALI — primitive type kullanılamaz
ArrayList<int> list = new ArrayList<int>();   // COMPILE ERROR!

// ✅ DOĞRU — wrapper class kullan
ArrayList<Integer> list = new ArrayList<Integer>();
ArrayList<String> names = new ArrayList<String>();

Generic Class Tanımlama — Sınav Projesi Şablonu

// Tek type parameter
public class Box<T> {
    private T value;
    public Box(T value) { this.value = value; }
    public T getValue() { return value; }
}

// İki type parameter — Dictionary ve University tam bu şekilde!
public class Dictionary<K, V> {
    private Map<K, V> map;

    public Dictionary() {
        map = new HashMap<K, V>();
    }

    public V get(K key) {
        if (!map.containsKey(key))
            throw new KeyNotFoundException(key);
        return map.get(key);
    }

    public void update(K key, V value) {
        map.put(key, value); // put: varsa günceller, yoksa ekler
    }
}

// Kullanım — farklı type kombinasyonları
Dictionary<String, String> d1 = new Dictionary<String, String>();
Dictionary<Integer, Double> d2 = new Dictionary<Integer, Double>();

Queue Implement Etmek İçin Hangi Yapılar Kullanılabilir?

Bütünleme sınavında "Which data structures could be used to implement a Queue?" diye soruldu:

  • ✅ ArrayList — index tabanlı, ön ve arka erişim mümkün
  • ✅ Array — circular array ile queue implement edilebilir
  • ✅ LinkedList — addLast/removeFirst ile FIFO
  • ❌ HashSet, HashMap, TreeSet, TreeMap — sıra kavramı yok
  • ❌ Stack — LIFO, Queue için uygun değil

3 Sets (Ch 21)

HashSet

Duplicate yok. Sırasız (hash tabanlı). add/remove/contains hepsi O(1).

Sıra önemli değilse

LinkedHashSet

Duplicate yok. Ekleme sırası korunur. HashSet'ten biraz yavaş.

Eklenme sırası lazımsa

TreeSet

Duplicate yok. Doğal sıralı (Comparable). add/remove/contains O(log n).

Sıralı tutmak gerekiyorsa

Set<String> set = new HashSet<String>();
set.add("apple");
set.add("banana");
set.add("apple");   // Duplicate — eklenmez
System.out.println(set.size());          // 2
System.out.println(set.contains("apple")); // true

for (String s : set) {
    System.out.println(s); // Sırasız çıkabilir
}

4 Maps (Ch 21) ⚡ Sınav Projesi İçin En Önemli

Map Türleri

HashMap<K,V>

Sırasız. put/get/containsKey O(1). Sınav projelerinde en sık kullanılan.

Genel kullanım

Ekleme sırası korunur. Dictionary projesindeki toString() çıktısında sıra önemli.

Sıra önemli

TreeMap<K,V>

Key'e göre sıralı. Comparable implementi gerekebilir. O(log n).

Key sırası lazım

Kritik Map Metodları — Sınav Projesinde Bunlar Kullanılıyor

Map<String, Integer> map = new HashMap<String, Integer>();

// Ekleme / Güncelleme (ikisi için de put kullanılır!)
map.put("Ali", 90);    // yeni ekle
map.put("Ali", 95);    // güncelle — eski değer üstüne yazar

// Okuma
map.get("Ali");        // 95
map.containsKey("Ali"); // true
map.containsKey("Veli"); // false

// Key ve Value koleksiyonları
Set<String> keys = map.keySet();          // tüm keyler
Collection<Integer> values = map.values(); // tüm değerler

// Iterate — Map.Entry
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}

// toString formatı sınav projesinde istenen format: {K=V, K=V}
System.out.println(map); // {Ali=95} — HashMap'in built-in toString'i bu formatı verir!
💡 Sınav İpucu: Dictionary projesinin toString() metodunda "{K=V, ...}" formatı isteniyor. HashMap'in kendi toString() metodu tam bu formatı üretiyor! Yani sadece return map.toString(); yazabilirsin.

Nested Map — University Projesinin Kalbi

// University<S, C, G> — öğrenci → (ders → not) haritası
// S = öğrenci tipi, C = ders tipi, G = not tipi
Map<S, Map<C, G>> studentCourseGrade;

// Örnek: Long student ID → String course → Integer grade
Map<Long, Map<String, Integer>> data = new LinkedHashMap<>();

// register: öğrenci yoksa yeni map ekle, dersi null ile koy
public void register(S student, C course) {
    data.putIfAbsent(student, new LinkedHashMap<>());
    data.get(student).put(course, null); // not henüz yok
}

// grade: öğrenci yoksa StudentNotFoundException, ders yoksa CourseNotFoundException
public void grade(S student, C course, G grade) {
    if (!data.containsKey(student)) throw new StudentNotFoundException(student);
    if (!data.get(student).containsKey(course)) throw new CourseNotFoundException(course);
    data.get(student).put(course, grade);
}

// transcript: "N/A" göster eğer not null ise
public String transcript(S student) {
    StringBuilder sb = new StringBuilder();
    sb.append("Transcript for Student " + student + "\n");
    for (Map.Entry<C, G> e : data.get(student).entrySet()) {
        String grade = (e.getValue() == null) ? "N/A" : e.getValue().toString();
        sb.append(e.getKey() + " - " + grade + "\n");
    }
    return sb.toString();
}

🧪 Pratik Sorular — Data Structures & Collections

S1: HashSet, LinkedHashSet ve TreeSet arasında ne fark var?
HashSet: sırasız. LinkedHashSet: ekleme sıralı. TreeSet: doğal sıralı (Comparable). Hepsi duplicate kabul etmez.
S2: Queue implementasyonu için hangi Java yapıları kullanılabilir? (Sınav sorusu!)
ArrayList, Array, LinkedList — HashSet/HashMap/Stack kullanılamaz.
S3: map.put("key", null) ne yapar?
Key'i ekler ama değeri null olarak tutar. containsKey() true döner. University projesinde register() yapılıp henüz not verilmemiş dersler için bu kullanılır (N/A göstermek için).
S4: Map<String, Integer> map = new HashMap<>(); — map.toString() ne döner?
{key1=val1, key2=val2} formatında String döner. Dictionary projesinin toString() için bunu doğrudan kullanabilirsin!
