1 Referans Semantiği: == vs .equals() ⚡ EN KRİTİK KONU

Hoca bu soruyu her sınavda soruyor. Fill-in-the-blanks bölümünün tamamı buna dayanıyor.

Temel Ayrım

  • == → iki değişkenin aynı bellek adresini (referansı) gösterip göstermediğini kontrol eder
  • .equals() → içeriği karşılaştırır (String için değer, custom class için override edilmemişse == gibi davranır)
String a = new String("CSE");   // Heap'te yeni obje
String b = new String("CSE");   // Heap'te başka yeni obje
String c = a;                   // Aynı referans

System.out.println(a == b);      // false — farklı adres
System.out.println(a.equals(b)); // true  — aynı içerik
System.out.println(a == c);      // true  — aynı adres
System.out.println(a.equals(c)); // true  — aynı içerik

⚠️ Sınav tuzağı: true / false çıktısını String ile elde etmek imkânsızdır. Aynı adres demek aynı içerik demektir String için. Bu output'u soran sorularda mutlaka custom class yazmak gerekir ve equals() metodu false döndürmelidir.

Custom Class ile true/false Üretmek

class MyClass {
    public int value;
    public MyClass(int value) { this.value = value; }

    // equals() override ediyoruz — her zaman false döndürüyor
    @Override
    public boolean equals(Object obj) {
        return false;
    }
}

// Kullanım:
MyClass r1 = new MyClass(1);
MyClass r2 = r1;              // Aynı referans → == true
System.out.println(r1 == r2);      // true
System.out.println(r1.equals(r2)); // false (override ettik)

2 Inheritance & Polymorphism (Ch 11)

Temel Kavramlar

  • extends — kalıtım için kullanılır: class Dog extends Animal
  • super() — parent class'ın constructor'ını çağırır
  • Override — parent'taki metodu child'da yeniden yazar (aynı imza)
  • Overload — aynı sınıfta aynı isimli metod, farklı parametre listesi
  • Polymorphism — parent tipindeki referans, child nesnesini tutabilir
🔴 En sık yanlış yapılan: Parent'ın no-arg constructor'ı yoksa, child class compiler'da hata verir. FinalExam → Exam örneği bütünlemede tam bu yüzden çıktı! 
class Exam {
    String name;
    public Exam(String name) { this.name = name; }
    // No-arg constructor YOK!
}

class FinalExam extends Exam {
    // ❌ COMPILE ERROR: super() çağırıyor ama Exam'da yok!
}

// ✅ ÇÖZÜM: FinalExam'a explicit constructor ekle
class FinalExam extends Exam {
    public FinalExam() {
        super("Final"); // parent'ın String constructor'ını çağır
    }
}

abstract class Shape {
    String color;                           // instance variable olabilir
    public Shape(String c) { color = c; }   // constructor olabilir
    public abstract double area();          // abstract method — body yok
    public String getColor() { return color; } // concrete method — body var
}

interface Drawable {
    int MAX = 100;                           // constant (public static final)
    void draw();                             // abstract method (implicit)
}

class Circle extends Shape implements Drawable {
    double radius;
    public Circle(double r, String c) { super(c); radius = r; }
    @Override public double area() { return Math.PI * radius * radius; }
    @Override public void draw() { System.out.println("Circle"); }
}

Interface: sadece constants ve abstract methodlar (Java 8 öncesi), birden fazla implement edilebilir.

Custom Exception — Sınav Projesi Şablonu

Her sınav projesinde custom exception istendiğini hatırla!

// Şablon — her sınav projesinde bu pattern kullanılıyor
class KeyNotFoundException extends RuntimeException {
    private Object key;

    public KeyNotFoundException(Object key) {
        this.key = key;
    }

    @Override
    public String toString() {
        return "KeyNotFoundException Key = " + key;
    }
}

// Kullanım:
public V get(K key) {
    if (!map.containsKey(key)) {
        throw new KeyNotFoundException(key);
    }
    return map.get(key);
}

// Yakalama:
try {
    dict.get("missing");
} catch (RuntimeException ex) {
    System.out.println(ex); // KeyNotFoundException Key = missing
}
💡 Hatırlatma: RuntimeException'dan extend edilen exception'lar unchecked — throws declaration zorunlu değil. Bütünleme projesinde StudentNotFoundException ve CourseNotFoundException da bu şablonu takip etti.

5 OOP Terminoloji Özeti — Eşleştirme Sorusu İçin

TerimTanım (sınavda verilen)
OverrideTwo methods with the same name, return type, and parameter list. One exists in a parent class but the other modifies the implementation.
OverloadTwo methods with the same name and return type but different parameter lists. Can be in the same class.
InterfaceA classlike construct that contains only constants and abstract methods.
Abstract ClassCannot create an Object from this class. Can contain abstract methods that will be defined in subclasses.
Non-Abstract ClassCan create an Object from this class. Cannot contain abstract methods.
